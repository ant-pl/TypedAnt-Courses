## `TypedAnt`代码风格

### 1. 缩进
- 统一使用 **Tab** 缩进
- 结构体、函数、if、while、for 内部必须缩进一层

### 2. 大括号 `{ }`
- 统一跟在语句同一行末尾

```ant
func func_name<T>() {
}

if a {
}
```

### 3. 命名
- 结构体：UpperCamelCase (PascalCase)
- 函数：snake_case
- 变量：snake_case
- 常量: UPPER_SNAKE_CASE
- 循环变量如果是数组索引可使用类似 *i* *j* *k* 等变量 (超过四层我这边还是建议你重写好吗)

### 4. 空格
- 运算符两边留空格：`a + b`、`i = i + 1`
- `sizeof T`、`as` 转换两边留空格
- 参数之间逗号后留空格
- 指针紧贴类型：`*T`、`*u8`，而不是 `* T`

### 5. 结构体
- 每个字段独占一行
- 冒号紧贴字段名
- 结构体名字 UpperCamelCase

### 6. 函数
- 泛型`<T>`紧跟函数名
- 参数列表整齐，类型写清楚
- 函数之间空一行分割（子函数与父函数除外 子函数若在父函数第一行则不空行）

### 7. 使用禁忌
- 不写完全看不懂的极简简写
